openapi_server.contract.eventListening.eventListenerSubject module

class openapi_server.contract.eventListening.eventListenerSubject.SmartContractEventListener(sc_contract)

Bases: openapi_server.contract.eventListening.abstractSubject.AbstractSubject

This class is responsible for Event listening of smart contract events.

Observers attach themselves to this event listener, which creates a thread each for the filtering of events. the update function of the affected observer is then called.

It follows the observer pattern as a guideline only, since notifyObserver is not used, as attaching an observer starts event listening immediately, and only a single event is to be updated.

The Smart Contract Event Listening using threads is based on the example of https://web3py.readthedocs.io/en/stable/filters.html#asynchronous-filter-polling
